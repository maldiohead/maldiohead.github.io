V8 漏洞分析-CVE-2019-5782

前言

该系列文章为学习浏览器安全的基础，目的是为了根据分析这些漏洞及其利用来学习浏览器漏洞。相对来说我会更多集中在漏洞的成因分析，以及漏洞利用的细节，所以对于操作过程可能会相对有所省略。

漏洞简述

该漏洞是因为数组优化所造成的数组越界访问导致的越界访问，漏洞品相不错，利用起来也是很好利用.

P0上的漏洞信息:

https://bugs.chromium.org/p/chromium/issues/detail?id=906043

漏洞存在的最新版本git hash:b474b3102bd4a95eafcdb68e0e44656046132bc9

可以看到第二个数组的大小变为65535，显然这一部分内存以及被篡改了。篡改的原因就是因为trubfan jit对fun函数调用的优化，产生了这个漏洞。

fun(…a3); // “…” convert array to arguments list

这个函数被优化调用，a3的长度是0x11000,x>>16*21 的值刚好是21 x>>16 *41的值是41

所以显然经过调用这个语句,数组a1越界访问，导致a2对象的结构体被覆盖。

上图中我们可以看到a1 a2的内存布局,a1[0]的地址是0x18b78f6055a8 a[21]的地址刚好是0x18b78f605650，此位置也就是a2.elements.lenght的长度。可以从debugptint 中看到。所以这也就是这个漏洞的核心。同理a1[41]也正好是a2.length长度的位置。
